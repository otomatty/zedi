# 作業日誌（Journal）

## 目的

- **1 日分**の作業内容を `today.md` に集約し、後から Zedi に転記・同期しやすくする。
- リポジトリには **長期アーカイブ用の `docs/` は置かない**方針のため、日誌はこのフォルダのみ使う。

## ファイル

| ファイル   | 役割                                                                                               |
| ---------- | -------------------------------------------------------------------------------------------------- |
| `today.md` | **当日**のメモ・完了タスク・次アクション。日付をまたいだら内容をリセットまたはアーカイブ先へ移す。 |

## Zedi への取り込み（定期実行の例）

1. 毎日決まった時刻に `journal/today.md` を読む。
2. Zedi のノート作成 API や UI に内容を貼る（プロジェクト固有のスクリプトは `scripts/` に追加してよい）。
3. 取り込み後、`today.md` を空にするか、日付見出しだけ残す。

エージェントは **`journal/` を自動では読まない**（必要なら `@journal/today.md` で添付する）。
