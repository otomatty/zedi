# ノート一覧・公開ノート（Discover）仕様

/notes のタブ構成、公開ノート一覧（Discover）、公式ノート・人気ノートの表示ルールを定義します。

---

## 1. /notes の構成

### 1.1 ルートとタブ

| パス              | 内容                                                                             | 備考                                                                                                                                                                                                   |
| ----------------- | -------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `/notes`          | ノート一覧ページ。**上部にタブ**で「参加中のノート」と「公開ノート」を切り替え。 | 従来どおり参加中のノートを表示。タブで Discover に切り替え可能。                                                                                                                                       |
| `/notes/discover` | 公開ノート（Discover）の表示。                                                   | タブで「公開ノート」を選んだときの表示。同一ページ内でタブ切り替えでも、URL は `/notes/discover` にしておく（または `/notes` のクエリ `?tab=discover` など。いずれにしてもタブで切り替え可能にする）。 |

**UI 方針**

- ページ上部に **タブ** を配置する。
  - **参加中のノート**（または「マイノート」）: 自分がオーナーまたはメンバーであるノートのみ表示。従来の /notes の挙動。
  - **公開ノート**（Discover）: `/notes/discover` に対応。公開ノート一覧（後述）。
- タブの切り替えで表示内容を切り替え、URL は「参加中」のとき `/notes`、「公開ノート」のとき `/notes/discover` にすると一覧で分かりやすい（推奨）。逆に `/notes?tab=discover` でも可。

### 1.2 参加中のノート（従来どおり）

- **表示対象**: 自分がオーナーであるノート、または note_members に含まれるノートのみ。
- **並び順**: **更新日時が新しい順**（`notes.updated_at` の降順）。
- ノート作成ボタンはこのタブ（参加中のノート）で表示する想定でよい。

---

## 2. 公開ノート（Discover）の表示

### 2.1 表示対象

- **visibility = 'public'** のノートのみを対象とする（unlisted は一覧に出さない）。
- **is_deleted = false** のノートのみ。

### 2.2 セクション構成

Discover ページは **2 セクション** に分ける。

1. **公式ノート**
   - 運営が「公式ノート」として指定したノート（`notes.is_official = true`）のみ。
   - 公式バッジを付けて表示。
   - 並び順: 更新日時が新しい順（updated_at 降順）など、運営側で決めてよい。

2. **公開ノート**（通常）
   - 公式ノート**以外**の public ノート（`is_official = false` または未設定）。
   - ここに「更新があったノート」「人気のノート」を表示する（後述）。
   - 公式ノートはこのセクションには**含めない**（除外する）。

### 2.3 通常の公開ノートの並び順・表示

- **更新があったノート**: **更新日時が新しい順**（`notes.updated_at` の降順）で表示する。
- **人気のノート**: **閲覧数**を基準とする。閲覧数が多い順に表示する。対象は「公式ノートを除いた public ノート」。
  - 人気順のみで一覧にするか、「更新順」と「人気順」の切り替え（タブやソート）を用意するかは実装で選択してよい。いずれにせよ「更新順」「人気順」の両方の考え方を満たすこと。

まとめ:

| セクション         | 対象                           | 並び順の基準                          |
| ------------------ | ------------------------------ | ------------------------------------- |
| 公式ノート         | is_official = true かつ public | 更新日時が新しい順（推奨）            |
| 公開ノート（通常） | public かつ 公式以外           | 更新日時が新しい順 / 閲覧数順（人気） |

---

## 3. 公式ノート

### 3.1 定義

- **公式ノート** = 運営（管理者）が「公式ノート」と指定したノート。
- 指定は **notes.is_official** フラグで管理する（true = 公式）。

### 3.2 誰が指定するか

- **運営（管理者）のみ**が、ノートを公式に指定・解除できる。
- 一般ユーザー（オーナー含む）は is_official を変更できない。管理画面または API で運営のみ実行可能とする。

### 3.3 表示

- 公式ノートには **公式バッジ** を付けて表示する（ノートカード・ノート詳細など）。
- Discover では **公式ノートセクション** を設け、通常の公開ノートとは**別**に表示する。公式ノートは「通常の公開ノート」セクションには含めない。

### 3.4 人気ノートからの除外

- **人気のノート** の集計・表示では、**公式ノートを除外**する。  
  閲覧数順の対象は「public かつ 公式以外」のノートのみとする。

---

## 4. 人気のノート（閲覧数）

### 4.1 定義

- **人気のノート** = **閲覧数**を基準にしたノート。
- 閲覧数 = そのノート（ノート詳細またはノート内ページ）が閲覧された回数。同一ユーザー・同一セッションの重複は実装方針に応じてカウントするか除外する（例: 1 ユーザー 1 日 1 回までなど）。

### 4.2 対象

- visibility = 'public' かつ **公式ノートではない**（is_official = false）ノートのみを対象とする。
- 公式ノートは人気順に含めない。

### 4.3 閲覧数の保持

- **notes.view_count**（整数、デフォルト 0）を用意する、または **note_views** のような閲覧ログテーブルを用意して集計する。
- 一覧でのソート・表示には、いずれかで「ノート単位の閲覧数」が取得できればよい。

---

## 5. データベース

### 5.1 notes テーブルへの追加

| カラム      | 型      | 説明                                                                            |
| ----------- | ------- | ------------------------------------------------------------------------------- |
| is_official | BOOLEAN | 公式ノートなら true。デフォルト false。運営のみ変更可。                         |
| view_count  | INTEGER | 閲覧数（ノート単位）。デフォルト 0。インクリメントはノート/ページ閲覧時に実施。 |

※ view_count の代わりに note_views テーブルで集計する場合は、notes には持たず、集計結果を API で返す。

### 5.2 マイグレーション（Aurora PostgreSQL）

マイグレーションファイル: `db/aurora/007_notes_official_and_view_count.sql`

```sql
ALTER TABLE notes ADD COLUMN is_official BOOLEAN NOT NULL DEFAULT FALSE;
ALTER TABLE notes ADD COLUMN view_count  INTEGER NOT NULL DEFAULT 0;
CREATE INDEX idx_notes_is_official ON notes(is_official);
```

## 6. URL とルート

| URL             | 画面                                                               | 認証                                       |
| --------------- | ------------------------------------------------------------------ | ------------------------------------------ |
| /notes          | ノート一覧（タブ: 参加中のノート \| 公開ノート）                   | 要ログイン                                 |
| /notes/discover | 公開ノート（Discover）。タブで「公開ノート」を選んだときと同じ内容 | 未ログインでも閲覧可。投稿はログイン必須。 |

- ルート定義では `/notes` で Notes ページを表示し、その中でパスが `/notes/discover` のとき（または子ルートで Discover を表示するとき）に Discover の内容を表示する。
- 実装パターン: `/notes` と `/notes/discover` を両方 Notes コンポーネント（または Notes レイアウト）で受け、パスに応じてタブのアクティブ状態と表示内容を切り替える。

---

## 7. まとめ

| 項目                 | 仕様                                                                                                                       |
| -------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| /notes               | 従来どおり「参加中のノート」を表示。上部にタブで「参加中のノート」と「公開ノート」を切り替え。                             |
| /notes/discover      | 公開ノート一覧。公式セクション ＋ 通常の公開ノート（更新順・人気順）。                                                     |
| 参加中のノートの並び | 更新日時が新しい順。                                                                                                       |
| 更新があったノート   | 更新日時が新しい順で表示。                                                                                                 |
| 人気のノート         | 閲覧数基準。公式ノートは除外。                                                                                             |
| 公式ノート           | 運営が公式と指定（is_official）。公式バッジ、Discover で公式セクションにのみ表示。通常の公開ノートセクションには出さない。 |

この仕様に合わせて、フロントのタブ・ルート・API（一覧取得・閲覧数・is_official）を実装します。
