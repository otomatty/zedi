# Polar 税込価格（税込み $20）の設定について

**作成日:** 2026-03-01  
**目的:** 「税込み $20 / $200」で表示・請求するために、Polar 側でどのように設定すべきかをまとめる。

---

## 調査結果サマリー

| 項目                             | 内容                                                                                                                                  |
| -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| **Polar のデフォルトの税の扱い** | 商品に設定した価格は **税抜**。チェックアウト時に購入者の所在地に応じて VAT/GST/消費税が **上乗せ** される。                          |
| **税込価格オプション**           | 公式ドキュメントには「価格を税込として扱う」ようなダッシュボード／API の設定は **記載なし**。                                         |
| **基盤 (Stripe Tax)**            | Polar は Stripe 上で動いており、Stripe には `tax_behavior`（inclusive / exclusive）があるが、Polar がその設定を露出しているかは不明。 |

## 0. 日本の価格表示ルール（総額表示義務）

**結論: 日本向けには「税込」を必ず分かるように表示する必要があります。税抜のみの表示は不可です。**

### 法令上の要件

- **消費税法**（総額表示義務）により、**課税事業者が消費者にあらかじめ価格を表示する場合、税込価格を表示することが義務**です（令和3年4月1日以降）。
- 対象: 店頭・チラシ・**インターネット広告・ECサイト**など、あらかじめ消費者に価格を見せる表示全般。

### 認められる表示の例（国税庁 No.6902）

いずれも **税込の金額が明瞭** であることが条件です。

| 表示例                       | 備考                            |
| ---------------------------- | ------------------------------- |
| 11,000円                     | 税込のみ                        |
| 11,000円（税込）             | 税込のみ                        |
| 11,000円（税抜価格10,000円） | **税込を主に、税抜を補足** → OK |
| 10,000円（税込価格11,000円） | **税抜を主に、税込を併記** → OK |

- **「税抜」だけを大きく表示し、税込が分からない**ような表示は **不適切** です。
- 税抜と税込の両方を出す場合は、**税込の金額を必ず明示**すれば、「税抜」表記を併用することは可能です。

### アプリでの推奨（日本ロケール）

- **推奨:** 主表示は **税込**（例: 「$22/月（税込）」または「約¥3,300/月（税込）」など、実際の支払いに近い金額）。
- **税抜も出したい場合:** 「$20（税抜）/ 税込 $22」のように、**税込を必ず併記**する。
- **避ける:** 「$20（税抜）」のみで、税込がどこにも書いていない表示。

参照: [国税庁 No.6902 総額表示の義務付け](https://www.nta.go.jp/taxes/shiraberu/taxanswer/shohi/6902.htm)

## 1. Polar ドキュメントで確認できた仕様

### 税の扱い（Merchant of Record）

- [Merchant of Record - Tax](https://polar.sh/docs/merchant-of-record/tax) より:
  - 例: **「$10/月のサブスクは、スウェーデン（VAT 25%）の消費者には $12.5/月になる」** と明記されている。
  - つまり **設定する価格 = 税抜** で、税が加算される前提。

### 料金計算の例（Fees ドキュメント）

- [Fees](https://polar.sh/docs/documentation/polar-as-merchant-of-record/fees) の例:
  - 商品価格: **$30**（税抜）
  - VAT (25%): **$7.50**
  - 合計: **$37.50**

→ Polar の説明では一貫して **「商品価格 = 税抜、税は上乗せ」** となっている。

## 2. 税込み $20 にするための選択肢

### 選択肢 A: Polar ダッシュボードで「税込」設定がないか確認する（推奨）

1. **Sandbox:** [https://sandbox.polar.sh/](https://sandbox.polar.sh/) にログイン
2. **Settings**（組織 or アカウント）を開く
3. 次のような項目がないか確認する:
   - Tax / VAT 関連の設定
   - 「Price includes tax」／「税込価格」のようなチェックやトグル
   - Stripe 連携や Tax の挙動（inclusive / exclusive）を選ぶオプション
4. **Products** の各商品編集画面でも、価格の横や詳細に「税込／税抜」の指定がないか確認する

**見つからない場合:** Polar サポートに問い合わせるのが確実。

- サポート: [https://polar.sh/support](https://polar.sh/support)
- 質問例:  
  「We want the customer to pay exactly $20/month including tax (tax-inclusive pricing). Is there a way to set product price as tax-inclusive, or should we set a lower base price so that after tax the total is $20 in specific regions?」

---

### 選択肢 B: 税抜価格を逆算して「税込 $20」に近づける

Polar に「税込」オプションがない場合、**主要市場の税率**を仮定して税抜価格を下げる方法。

- **目標:** 顧客の支払い総額が約 **$20/月**・**$200/年** になるようにする。
- **計算例（税率 10% の場合）:**
  - 税込 $20 → 税抜: \( 20 \div 1.10 \approx 18.18 \)
  - 税込 $200 → 税抜: \( 200 \div 1.10 \approx 181.82 \)
- Polar のプロダクトには **$18.18 / $181.82** のように設定する（または切り上げ・切り下げで運用しやすい金額にする）。

**注意:**

- 国・地域で税率が異なるため、**すべての国でちょうど $20 にはならない**。
- 日本 (10%)、EU (17–25%)、米国 (州ごと) などで支払い総額が変わる。
- 「主に想定する市場」を決めて、その税率で逆算する運用になる。

**手順（Polar 側）:**

1. [Sandbox](https://sandbox.polar.sh/) または [Production](https://polar.sh/) の **Products** を開く
2. **Zedi Pro (Monthly)** を編集 → **Price** を税抜希望額（例: $18.18 または $18）に変更
3. **Zedi Pro (Yearly)** を編集 → **Price** を税抜希望額（例: $181.82 または $182）に変更
4. 保存後、該当国でテスト決済して税込総額を確認

---

### 選択肢 C: 表示は「$20（税込）」のまま、Polar は $20 のままにする

- **Zedi の料金ページ:** すでに「$20/月（税込）」「$200/年（税込）」と表示している。
- **Polar のプロダクト価格:** 従来どおり **$20 / $200**（税抜）のままにする。

この場合の意味づけ:

- 「$20」は **目安の税込価格** として表示する。
- 実際の請求は **$20 + 地域の税** となり、例えば日本では約 $22 になる。
- 注記で「価格は税込表示です。地域により税が加算される場合があります」などと書くと誤解が減る。

**Polar 側の設定変更は不要。** 表示と文言の調整のみで対応できる。

## 3. 推奨アクション

| 優先度 | アクション                                                                                                    |
| ------ | ------------------------------------------------------------------------------------------------------------- |
| 1      | Polar ダッシュボード（Settings / Products）で「税込」「Tax inclusive」などのオプションを確認する              |

| 4      | 当面は **選択肢 C** のまま（表示のみ $20 税込）で運用し、Polar の回答や今後の機能に合わせて B や A を採用する |

## 5. 本プロジェクトでの表示まわり（現状）

- 料金表示は i18n で **$20 / $200**、注記は **「/ 月（税込）」「/ 年（税込）」** にしている（`src/i18n/locales/ja/pricing.json`, `en/pricing.json`）。
