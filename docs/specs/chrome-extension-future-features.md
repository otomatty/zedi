# Chrome拡張 将来機能と導入順序

**日付**: 2026-03-15  
**目的**: zedi 連携 Chrome 拡張の将来機能を「即時処理」と「非同期ジョブ」に分類し、導入順序を定義する。

---

## 1. 分類の原則

| 分類             | 定義                                                     | 例                                             |
| ---------------- | -------------------------------------------------------- | ---------------------------------------------- |
| **即時処理**     | ユーザー操作に対し、数秒以内に完了しレスポンスを返す処理 | クリップ保存、選択範囲保存、既存ノート候補取得 |
| **非同期ジョブ** | 時間がかかるため即時応答せず、後追いで反映する処理       | AI 要約、タグ提案、重複検知                    |

拡張 UI は「最小操作を維持する」原則に従い、重い処理はサーバー / ジョブに委譲する。

---

## 2. 将来機能の一覧と分類

### 2.1 クリップ高度化

| 機能                             | 分類 | 説明                                               | 導入フェーズ            |
| -------------------------------- | ---- | -------------------------------------------------- | ----------------------- |
| URL 保存（現行 clip-and-create） | 即時 | URL から HTML 取得 → Readability 抽出 → ページ作成 | Phase 2（実装済み計画） |
| 選択範囲保存                     | 即時 | ページ内のテキスト選択範囲を新規ページに保存       | Phase 2+                |
| スクショ + OCR                   | 即時 | 画面キャプチャ → OCR テキスト → ページ作成         | Phase 3                 |
| 動画タイムスタンプ付きメモ       | 即時 | YouTube 等の動画 URL + タイムスタンプをメモに保存  | Phase 3                 |

### 2.2 コンテキスト連携

| 機能                 | 分類 | 説明                                                 | 導入フェーズ |
| -------------------- | ---- | ---------------------------------------------------- | ------------ |
| 現在タブ情報の取得   | 即時 | タブの URL / タイトルを API に渡して保存先候補を取得 | Phase 2+     |
| 既存ノート候補の提示 | 即時 | 現在のページ内容と関連しそうなノートを検索して提案   | Phase 2+     |

### 2.3 インテリジェント処理（非同期）

| 機能           | 分類   | 説明                                                 | 導入フェーズ |
| -------------- | ------ | ---------------------------------------------------- | ------------ |
| 要約           | 非同期 | 保存後に AI で要約を生成し、ページに追記             | Phase 3      |
| タグ提案       | 非同期 | 保存後にコンテンツからタグを提案し、候補を UI に反映 | Phase 3      |
| 重複検知       | 非同期 | 保存後に類似ページを検索し、重複候補を通知           | Phase 3      |
| 関連ノート提示 | 非同期 | 保存後に関連ノートを検索し、リンク候補を追加         | Phase 4      |

### 2.4 コラボ連携

| 機能                     | 分類 | 説明                                          | 導入フェーズ |
| ------------------------ | ---- | --------------------------------------------- | ------------ |
| 共有先・プロジェクト選択 | 即時 | 保存時にノート / プロジェクトを選択して即反映 | Phase 3      |

### 2.5 オフライン耐性

| 機能           | 分類 | 説明                                                 | 導入フェーズ |
| -------------- | ---- | ---------------------------------------------------- | ------------ |
| 一時キュー保存 | 即時 | オフライン時にキューに追加し、オンライン復帰時に同期 | Phase 4      |

---

## 3. 導入順序サマリ

```
Phase 2:   clip-and-create（URL 保存）、OAuth 認証
Phase 2+:  選択範囲保存、タブ情報・既存ノート候補（コンテキスト連携）
Phase 3:   スクショ + OCR、動画タイムスタンプ、共有先選択、要約・タグ提案・重複検知
Phase 4:   関連ノート提示、オフラインキュー
```

---

## 4. 技術的方針

- **即時処理**: 拡張専用 API の各 capability に対応するエンドポイントで同期レスポンス
- **非同期ジョブ**: 即時応答で `job_id` を返し、WebSocket または polling でジョブ完了を通知。拡張は完了後に必要に応じて UI を更新（オプション）
