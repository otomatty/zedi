# クイックツアー 仕様書

## 1. 概要

### 1.1 目的

初回利用者や初期設定ウィザードで「クイックツアーを見る」を選んだユーザーに対し、ホーム画面を中心に **主要な UI と操作を短いステップで案内** し、アプリの使い方を理解してもらう。

### 1.2 スコープ

- **対象画面:** ホーム（`/home`）のみ。ノート一覧・ページエディタ・設定画面は今回の対象外とする。
- **対象ユーザー:** サインイン済み。ツアーは「初期設定ウィザード完了後にツアーを選択した場合」または「後から設定等で再実行した場合」に表示する。
- **言語:** 既存の i18n（`ja` / `en`）に従い、ツアー文言も翻訳キーで管理する。

---

## 2. ユーザー体験

### 2.1 ツアー開始のトリガー

| トリガー                                                   | 説明                                                                                                        |
| ---------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------- |
| 初期設定ウィザード Step 3 で「クイックツアーを見る」を選択 | ウィザード完了 → `/home` に遷移し、`location.state.startTour === true` を検知してツアーを開始（既存実装）。 |
| 設定画面からの「ツアーを再実行」（将来）                   | 任意。実装時は「一般設定」等に「クイックツアーをもう一度見る」を追加し、`startTour()` を呼ぶ。              |

### 2.2 ツアー中の挙動

- **表示場所:** ホーム画面上にオーバーレイ（スポットライト＋吹き出し）で表示。
- **進行:** ステップごとに「次へ」で進む。最後のステップで「完了」を押すとツアー終了。
- **中断:** 「スキップ」またはオーバーレイ外クリックでツアーを終了できる。終了時も **ツアー完了として記録** する（`markTourCompleted()`）。
- **ページ遷移:** ツアー中はホームから離れない想定。万が一ユーザーが別ページへ遷移した場合はツアーを終了し、次回アクセス時には自動では再開しない。

### 2.3 ツアー完了後

- `markTourCompleted()` を呼び、`onboardingState.hasCompletedTour === true` を保存する。
- オーバーレイを閉じ、通常のホーム画面に戻る。
- 以降、同じデバイス（localStorage）では「初回ツアー」としては自動表示しない。再実行は設定等から明示的に行う。

---

## 3. ツアーステップ（案）

ホーム画面の要素に順にフォーカスし、短い説明を行う。ステップ数は **4〜6 程度** を目安とする。

| 順  | 対象要素                                         | 内容（日本語例）                                                          | 備考                                                                                             |
| --- | ------------------------------------------------ | ------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| 1   | ページグリッド（または「ホーム」の説明）         | ここにあなたのページが並びます。1ページ1アイデアで書いていきましょう。    | メインコンテンツエリア。要素が空の場合は「ページがまだない」旨を補足してもよい。                 |
| 2   | ヘッダー内のアプリメニュー（グリッドアイコン）   | ホームやノートなど、アプリ間の切り替えはここから。                        | `AppsMenu` のトリガーボタン。                                                                    |
| 3   | グローバル検索の説明（キーボードショートカット） | Ctrl+K（Mac は Cmd+K）でいつでも検索できます。                            | 検索 UI はショートカットで開くため、画面上の「検索ボタン」が無い場合は文言のみのステップにする。 |
| 4   | 右下の FAB（＋ボタン）                           | 新しいページは右下の ＋ から。空白ページ・URLから・画像から作成できます。 | `FloatingActionButton`。ツアーのハイライト対象として重要。                                       |
| 5   | （オプション）リンクのヒント                     | ページ内で [[ と入力すると、他のページへリンクできます。                  | テキストのみのステップでも可。                                                                   |
| 6   | 完了                                             | 準備できました。さっそくページを作ってみましょう。                        | 最後のステップ。「完了」でツアー終了。                                                           |

- **対象要素の指定:** 各ステップで「どの DOM をハイライトするか」をセレクタで指定する。実装時に `data-tour-id="home-page-grid"` などの属性を付与し、ライブラリから参照する。
- **要素が存在しない場合:** 例）ページが 0 件で PageGrid が EmptyState の場合、ハイライトせず説明文だけ表示する、または「このエリアにページが並びます」のようにする。

## 4. 技術方針

### 4.1 ライブラリ

- **推奨:** [react-joyride](https://github.com/gilbarbara/react-joyride) を使用する。
  - ステップ配列で「target（セレクタ）・content・placement」を定義できる。
  - 継続／スキップ／完了のコールバックで `markTourCompleted()` や state 更新が可能。
  - アクセシビリティ（ARIA）への配慮がなされている。
- **代替:** Driver.js、Intro.js 等でも同様の要件は満たせる。既存スタイルやバンドルサイズに合わせて選定する。

### 4.2 既存コードとの接続

- **useOnboarding（`src/hooks/useOnboarding.ts`）**
  - `startTour()`: ツアー用 state（例: `isTourRunning`）を true にし、Joyride の `run: true` に紐づける。現在は `// TODO: Trigger tour start` のままなので、ここでツアー開始処理を実装する。
  - `completeTour()`: 既存の `markTourCompleted()` を呼ぶ。ツアー完了時・スキップ時に呼ぶ。
- **Home（`src/pages/Home.tsx`）**
  - `location.state?.startTour === true` のときに `startTour()` を呼ぶ処理は既にある。ツアー開始の実体（Joyride のマウントと `run` の制御）を Home または共通の Tour コンポーネントで行う。
- **onboardingState（`src/lib/onboardingState.ts`）**
  - `hasCompletedTour` と `markTourCompleted()` は既存のまま利用。ツアー完了／スキップのいずれでも `markTourCompleted()` を 1 回だけ呼ぶ。

### 4.3 セレクタ用の data 属性

ツアーでハイライトする要素には、以下のような `data-tour-id` を付与する（例）。

| data-tour-id          | 想定要素                                          |
| --------------------- | ------------------------------------------------- |
| `tour-home-page-grid` | PageGrid のコンテナ、またはメインコンテンツエリア |
| `tour-apps-menu`      | AppsMenu のトリガーボタン                         |
| `tour-fab`            | FloatingActionButton のメインボタン               |

- グローバル検索はショートカットで開くため、画面上に常時表示される要素が無い場合は「ターゲットなし」のテキストステップとする。
- セレクタは `[data-tour-id="..."]` のように属性セレクタで指定する。

### 4.4 多言語

- ツアーのタイトル・本文は i18n で管理する。
- 新規 namespace 例: `tour`。`src/i18n/locales/ja/tour.json` と `en/tour.json` を追加し、`steps[].title` / `steps[].content` をキーで参照する。
- キーボードショートカット（Ctrl+K / Cmd+K）は言語ごとに文言を切り替える（例: `tour.shortcutSearch` で「Ctrl+K（Mac は Cmd+K）で検索」）。

## 5. 状態と永続化

- **hasCompletedTour:** localStorage（`zedi-onboarding`）に保存。一度 true になれば、同じブラウザでは初回ツアーとしては自動表示しない。
- **ツアー再実行:** 設定画面等から「クイックツアーをもう一度見る」を選んだ場合は、`hasCompletedTour` の有無に関わらず `startTour()` を呼び、ツアーを表示してよい。
- **テスト用:** 既存の `resetOnboardingState()` でオンボーディング状態をリセットすると、ツアー未完了扱いになる。

---

## 6. アクセシビリティ

- ツアー UI はキーボードで操作可能にする（次へ・戻る・スキップ・閉じる）。
- スポットライトの背後にある操作は、ツアー中は無効化するか、スキップで閉じたあと操作可能にすることが望ましい。
- スクリーンリーダー向けに、現在のステップ番号や「ツアーです」といった説明を読み上げられるようにする（react-joyride のデフォルトの挙動を確認し、不足があれば aria 属性を補う）。

---

## 7. 今後の拡張（スコープ外）

- ノート一覧・ページエディタ・設定画面用の別ツアーやステップの追加。
- ステップごとの「完了」記録（`completedSteps`）を利用した、途中再開や部分的スキップ。
- ツアー中の「前へ」ボタン（オプション）。

## 8. まとめ

| 項目       | 内容                                                                                          |
| ---------- | --------------------------------------------------------------------------------------------- |
| 目的       | ホーム画面の主要 UI を短いステップで案内する                                                  |
| 対象画面   | `/home` のみ                                                                                  |
| トリガー   | 初期設定ウィザードで「クイックツアーを見る」選択時（および将来の設定からの再実行）            |

| 実装       | react-joyride 推奨。`useOnboarding.startTour()` で開始、完了/スキップで `markTourCompleted()` |
| 多言語     | `tour` 用の i18n キーを追加（ja/en）                                                          |
| セレクタ   | `data-tour-id` を対象要素に付与し、`[data-tour-id="..."]` で指定                              |

この仕様に沿ってクイックツアーを実装する。
