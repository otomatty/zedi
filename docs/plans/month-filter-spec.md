# ヘッダー年月選択・ホーム月フィルタ 仕様書

## 1. 現在の実装状況

### 1.1 ヘッダー（MonthNavigation）

| 項目 | 内容                                                                                                        |
| ---- | ----------------------------------------------------------------------------------------------------------- |
| 場所 | `src/components/layout/Header/MonthNavigation.tsx`                                                          |
| UI   | 現在月を「yyyy年M月」で表示し、左右に前月・次月ボタン（ChevronLeft / ChevronRight）                         |
| 状態 | `useState` で `currentMonth`（Date）を保持している**のみ**                                                  |
| 問題 | ヘッダーの状態が **どこからも参照されていない**。前月・次月を押してもホームのページ一覧には一切反映されない |

### 1.2 ホーム・ページ一覧（PageGrid）

| 項目             | 内容                                                                  |
| ---------------- | --------------------------------------------------------------------- |
| 場所             | `src/pages/Home.tsx` → `src/components/page/PageGrid.tsx`             |
| データ           | `usePagesSummary()` で全ページを取得（StorageAdapter → IndexedDB）    |
| 表示             | 削除以外のページを `updatedAt` 降順でソートし、**すべて**グリッド表示 |
| フィルタ         | なし（月による絞り込みなし）                                          |
| ページネーション | なし（一覧は全件表示）                                                |

### 1.3 データ・インフラ

- **PageSummary**（`src/types/page.ts`）に `updatedAt: number`（ミリ秒タイムスタンプ）あり。
- リポジトリは `getPagesSummary(userId)` で**全件**返却。月指定の API はないため、**クライアント側でのフィルタ**が前提。
- `dateUtils.ts` に `formatMonthYear`・`getDateKey` あり。月範囲の判定用に `startOfMonth` / `endOfMonth`（date-fns）の利用が可能。

### 1.4 既存 UI パーツ

- **DropdownMenu**: `src/components/ui/dropdown-menu`（AppsMenu で使用済み）→ 年月選択のドロップダウンに流用可能。
- **Pagination**: `src/components/ui/pagination.tsx` に Pagination, PaginationPrevious, PaginationNext 等あり。ホームでは未使用。

## 2. 推奨仕様（UX を優先した案）

### 2.1 期間の考え方

- **「全期間」**: 従来どおり、更新日の制限なしで全ページを表示。**デフォルト**とする。
- **「yyyy年M月」**: その月の 00:00:00 ～ 23:59:59.999（**ローカルタイム**）に `updatedAt` が含まれるページのみ表示。

### 2.2 期間状態の持ち方（推奨: URL）

| 方式              | メリット                                     | デメリット                 |
| ----------------- | -------------------------------------------- | -------------------------- |
| **URL クエリ**    | 共有・ブックマーク・ブラウザ戻るで再現できる | 実装がやや増える           |
| Context / Zustand | 実装が簡単                                   | 共有不可・リロードで消える |

**推奨**: **URL クエリ**で持つ。

- 例: `/home` → 全期間
- 例: `/home?month=2025-02` → 2025年2月に更新されたページのみ

これにより「この月の一覧」をリンクで共有でき、戻る/進むでも期間が復元される。

### 2.3 ヘッダー UI（MonthNavigation）

1. **表示ラベル**
   - 期間が「全期間」のとき: **「全期間」** と表示。
   - 特定月のとき: **「yyyy年M月」**（既存の `formatMonthYear` と合わせる）。

2. **クリックでドロップダウン**
   - トリガー: いまの「yyyy年M月」または「全期間」のラベル部分をクリックで開く。
   - ドロップダウン内容（案）:
     - 先頭に **「全期間」** を常に表示。
     - 続いて **月の一覧**。次のいずれかが現実的:
       - **A) データがある月だけ**
         - 現在の `usePagesSummary` から「更新が一度でもあった月」を算出し、新しい順で表示。
         - メリット: 空の月を選べず迷いにくい。
       - **B) 直近 12 ヶ月 + 今月**
         - 実装が簡単で、どの月でも開ける。
   - 推奨: **A（データがある月のみ）** を優先し、該当月が 0 件のときは「この月に更新されたページはありません」を表示。

3. **前月・次月ボタン**
   - **全期間のとき**: 押したら「その月」を選択してドロップダウンを閉じ、URL を `?month=yyyy-MM` に更新（ラベルは「yyyy年M月」に切り替え）。
   - **特定月のとき**: 前月/次月に切り替え、URL と表示を更新。
   - ボタンは現状どおり残し、ラベルクリックと併用できるようにする。

4. **アクセシビリティ**
   - ドロップダウンはキーボード操作可能に（既存 DropdownMenu の仕様に従う）。
   - トリガーに `aria-label`（例: 「表示期間を選択」）を付与。

### 2.4 ホーム・ページ一覧（PageGrid）

1. **期間の参照**
   - Home または PageGrid で URL の `month` を読む（例: `useSearchParams()`）。
   - `month` なし or `all` → 全期間。
   - `month=2025-02` → 2025年2月でフィルタ。

2. **フィルタロジック**
   - 全期間: 現状どおり `updatedAt` 降順の全件（削除除く）。
   - 特定月:
     - その月のローカルでの「月初 00:00:00」と「月末 23:59:59.999」のタイムスタンプを算出。
     - `page.updatedAt` がこの範囲内のページだけ表示。
   - ソートは従来どおり `updatedAt` 降順。

3. **ページネーション**
   - **対象**: フィルタ後の一覧（全期間でも特定月でも同じ）。
   - 1 ページあたり **20 件または 24 件**（グリッドの列数に合わせて 4 の倍数が扱いやすい）。
   - 既存の `Pagination` コンポーネントを使用し、前へ・次へ・現在ページ表示を行う。
   - 1 ページに収まる件数ならページネーションは非表示でよい。

### 2.5 空状態

- 特定月を選んだ結果、**0 件**のとき:
  - メッセージ例: **「この月に更新されたページはありません」**。
  - オプション: 「全期間を表示」などのリンクで `?month=` を外し、全期間に戻す。

### 2.6 多言語（i18n）

- 追加したいキー例（en/ja）:
  - `period.all` → 「全期間」 / "All time"
  - `period.selectMonth` → 「表示期間を選択」 / "Select period"
  - `monthFilter.empty` → 「この月に更新されたページはありません」 / "No pages updated in this month."
  - `monthFilter.showAll` → 「全期間を表示」 / "Show all"
  - 必要に応じて `pagination.previous` / `pagination.next`（既存 Pagination が "Previous"/"Next" なら、日本語は「前へ」「次へ」など）

## 3. 実装時の注意点

- **タイムゾーン**: `updatedAt` は数値（ms）のため、月初・月末の範囲は **ユーザーのローカルタイム** で計算する（`date-fns` の `startOfMonth` / `endOfMonth` を `new Date(ts)` に適用）。
- **URL とヘッダーの同期**: ヘッダーは「URL を読んで表示」「選択時に URL を更新」の一方向にすると、Home とヘッダーで不整合が起きにくい。
- **MonthNavigation の表示場所**: ヘッダーは Home 以外（Notes 等）でも使われるため、**ホームでだけ**「期間フィルタ」が効くようにする。Notes では月選択は無視するか、同じヘッダーでも月選択を出さない、あるいは出しても Home に遷移したときに効く、などの方針を決める。推奨は「月選択はホーム専用で、URL は `/home?month=...` のみで解釈する」とし、他画面ではラベルだけ「全期間」固定でもよい。

## 4. まとめ

| 項目             | 内容                                                                                                        |
| ---------------- | ----------------------------------------------------------------------------------------------------------- |
| デフォルト表示   | 「全期間」で全ページを表示                                                                                  |
| ヘッダー UI      | ラベルクリックでドロップダウン → 「全期間」＋月一覧（データがある月を推奨）。前月・次月ボタンで月を切り替え |
| 状態管理         | URL クエリ `?month=yyyy-MM`（省略時は全期間）                                                               |
| 一覧             | 対象月に `updatedAt` が含まれるページのみ表示（クライアント側フィルタ）                                     |
| ページネーション | フィルタ後の一覧を 20〜24 件/ページで表示（既存 Pagination を使用）                                         |
| 空状態           | その月に 0 件のときメッセージ＋「全期間を表示」を案内                                                       |

この仕様にすると、**共有・戻る操作・空の月での案内・キーボード操作**まで含めた UX を揃えられます。実装順序の提案は次のとおりです。

1. URL と期間フィルタ用のユーティリティ（`month` のパース・月初月末の算出）。
2. MonthNavigation の「全期間」表示＋ドロップダウン＋前月・次月で URL 更新。
3. PageGrid で URL を参照して月フィルタ＋ページネーション導入。
4. 空状態のメッセージと i18n の追加。
