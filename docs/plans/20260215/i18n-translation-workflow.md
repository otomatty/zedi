# 翻訳（i18n）作業の進め方

**作成日:** 2026-02-15  
**目的:** 翻訳ファイルの粒度と、今後の翻訳作業の順番・ルールを整理する。

---

## 1. 現在の翻訳ファイル構成

翻訳は**ドメイン（名前空間）ごとにファイル分割**し、1ファイルあたりのコード量を抑えている。

```
src/i18n/
├── index.ts                    # 各 JSON を読み込みマージして i18n に渡す
└── locales/
    ├── ja/
    │   ├── common.json         # 共通（保存・キャンセル・読み込み中など）
    │   ├── settings.json       # 設定トップ・AI設定・ストレージ設定の見出し
    │   ├── generalSettings.json # 一般設定画面の全文言
    │   └── nav.json            # ヘッダー・ユーザーメニュー（サインイン・設定・プラン等）
    └── en/
        ├── common.json
        ├── settings.json
        ├── generalSettings.json
        └── nav.json
```

- **キー階層**: コンポーネントでは `t("common.save")` や `t("generalSettings.profile.title")` のように、トップレベルが「ファイル名（名前空間）」に対応する。
- **追加時**: 新ドメインが増えたら `locales/ja/<名前空間>.json` と `locales/en/<名前空間>.json` を追加し、`index.ts` で import して `ja` / `en` にマージする。

---

## 2. 名前空間の分け方（粒度の目安）

| 方針 | 説明 |
|------|------|
| **1画面 or 1機能 = 1名前空間** | 例: 一般設定 → `generalSettings`、設定トップ → `settings`。 |
| **共通語は common** | 保存・キャンセル・エラー・読み込み中など、複数画面で使う短い文言は `common` にまとめる。 |
| **ナビ・グローバルは nav** | ヘッダー、フッター、ユーザーメニューなど、どの画面でも出る文言は `nav`。 |
| **1ファイルは 50〜80 キー程度を目安** | これ以上増える場合は、サブ画面やサブ機能ごとに名前空間を分割する（例: `aiSettings`, `storageSettings`）。 |

新規追加する名前空間の例:

- `editor` … エディタ内（プレースホルダー、ツールチップ、ボタンラベルなど）
- `aiSettings` … AI設定フォーム専用（既存の `settings.ai` を移行しても可）
- `storageSettings` … 画像ストレージ設定フォーム専用
- `home` … ホーム画面
- `notes` … ノート一覧・ノート関連
- `auth` … サインイン・認証コールバック

---

## 3. 今後の翻訳作業の推奨順序

既存の日本語文字列を段階的にキー化し、英語リソースを追加する場合の順番。

| 順番 | 対象 | 内容 | 優先度の理由 |
|------|------|------|----------------|
| 1 | **設定まわり** | 設定トップ・一般設定・AI設定・ストレージ設定 | すでに `settings` / `generalSettings` は対応済み。AI・ストレージは文言が多いので `aiSettings` / `storageSettings` に分割して進めるとよい。 |
| 2 | **ナビ・共通** | ヘッダー、サイドメニュー、共通ボタン・ラベル | 全画面で見えるため、一度やれば体感の変化が大きい。`nav` と `common` は済んでいるので、他グローバル文言の洗い出し。 |
| 3 | **認証** | サインイン・認証コールバック・エラーメッセージ | 未ログインでも表示されるため、多言語対応の印象が強い。`auth` 名前空間を新設。 |
| 4 | **ホーム・ノート一覧** | Home, Notes のタイトル・空状態・説明文 | ログイン直後に見る画面。`home` / `notes` を追加。 |
| 5 | **エディタ** | プレースホルダー・ツールバー・スラッシュコマンド・画像アップロード等 | 文言量が多いので、`editor` をさらに `editor.toolbar`, `editor.placeholder` のようにサブで分けてもよい。 |
| 6 | **ノート詳細・共有** | NoteView, NoteSettings, NoteMembers, 共有まわり | ノート機能の文言を `notes` または `noteDetail` などにまとめる。 |
| 7 | **その他** | Pricing, Donate, Landing, NotFound 等 | 公開ページ・エラー系。必要に応じて `pricing`, `donate`, `landing`, `errors` など。 |

進め方のコツ:

- **1名前空間ずつ完了させる**: その画面の日本語をすべてキー化し、対応する英語を入れてから次へ。
- **キー名は意味でつける**: `buttonSave` より `common.save` のように、役割が分かる名前にする。
- **長文はキーを分ける**: 1キーに段落ごと入れず、見出し・本文・ボタンなどに分けると翻訳しやすい。

---

## 4. 新規名前空間の追加手順

1. **JSON を追加**  
   `src/i18n/locales/ja/<名前空間>.json` と `src/i18n/locales/en/<名前空間>.json` を作成。
2. **index.ts で読み込み**  
   `import jaXxx from "./locales/ja/xxx.json"` および `en` を追加。
3. **マージオブジェクトに追加**  
   `ja` / `en` のオブジェクトに `xxx: jaXxx` を追加。
4. **コンポーネントで使用**  
   `t("xxx.someKey")` のように参照。

既存の `settings` を分割する場合（例: AI設定を `aiSettings` に移す）:

- `settings.json` から該当キーを削除し、`aiSettings.json` に移動。
- コンポーネント内の `t("settings.ai.xxx")` を `t("aiSettings.xxx")` に変更。
- 英語側も同様に移す。

---

## 5. まとめ

| 項目 | 内容 |
|------|------|
| ファイル粒度 | ドメインごとに分割（common / settings / generalSettings / nav など）。1ファイル 50〜80 キー目安。 |
| キー設計 | トップレベル = 名前空間（ファイル名）。`t("名前空間.サブ.キー")` で参照。 |
| 翻訳の順番 | 設定 → ナビ・共通 → 認証 → ホーム・ノート一覧 → エディタ → ノート詳細 → その他。 |
| 新規追加 | `ja/` と `en/` に JSON を追加し、`i18n/index.ts` で import してマージ。 |

この順番と粒度に従うと、翻訳の抜け漏れが減り、ファイルも肥大化しにくい。
