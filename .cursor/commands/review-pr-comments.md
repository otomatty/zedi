# PR レビューコメントの分析

対象 PR: $ARGUMENTS

## 手順

1. **コメントの取得**
   セッション中に既にコメントを取得済みならそれを使う。
   未取得の場合は `/fetch-pr-comments` と同じ方法で取得する。

2. **コメントごとの分析**
   各コメントについて以下を行う：
   - 指摘箇所の現在のコードを確認する
   - 指摘が技術的に正しいか検証する
   - プロジェクトの規約（ESLint ルール含む）と照合する

3. **判断基準**

   **対応する** — バグ、セキュリティリスク、型安全性の不備、規約違反
   **対応しない** — 技術的誤り、ESLint 制約で不可、PR スコープ外、既に対処済み

4. **結果の報告**

   | #   | ファイル | 指摘内容 | 判断        | 理由 |
   | --- | -------- | -------- | ----------- | ---- |
   | 1   | ...      | ...      | 対応 / 不要 | ...  |

   ユーザーの承認後に修正に進む。
