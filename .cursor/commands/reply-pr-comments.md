# PR レビューコメントへの返信

対象 PR: $ARGUMENTS

## 手順

1. **情報の収集**
   まず `$ARGUMENTS` があれば PR URL / 番号として解決する。
   `$ARGUMENTS` が空で、セッション中に既に PR 情報やコメントを取得済みならそれを使う。
   どちらもない場合:
   - ブランチから PR を自動検出
   - 未返信のレビューコメントを取得
   - `git log` で対応コミットを確認

2. **各コメントへの対応状況を分析**
   - コメントが指摘した箇所の現在のコードを確認
   - 対応コミットがあればその差分を特定

3. **返信内容の作成**

   **修正した場合：**

   ```
   修正しました。{修正内容の要約}（{コミットSHA}）
   ```

   **対応しない場合：**

   ```
   ご指摘ありがとうございます。{対応しない理由}
   ```

4. **投稿前の確認**
   全返信を一覧で表示し、**ユーザーの承認を得てから投稿する**:

   | #   | コメント要約 | 対応        | 返信内容 |
   | --- | ------------ | ----------- | -------- |
   | 1   | ...          | 修正 / 不要 | ...      |

5. **返信の投稿**
   ```bash
   gh api -X POST repos/<owner>/<repo>/pulls/<番号>/comments/<comment_id>/replies \
     -f body="返信内容"
   ```
