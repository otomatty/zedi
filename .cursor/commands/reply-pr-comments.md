# PR レビューコメントへの返信

対象 PR: $ARGUMENTS

## 手順

1. **情報の収集**
   - `gh pr view <番号> --repo <owner/repo>` で PR の概要を取得
   - `gh api repos/<owner>/<repo>/pulls/<番号>/comments` でレビューコメント一覧を取得
   - `gh pr diff <番号> --repo <owner/repo>` で最新の差分を取得
   - `git log` で対応後のコミット履歴を確認

2. **各コメントへの対応状況を分析**
   未返信のレビューコメントごとに：
   - コメントが指摘した箇所の現在のコードを確認
   - 対応コミットがあればその差分を特定
   - 対応の有無と内容を判断

3. **返信内容の作成**
   各コメントに対して以下のいずれかのパターンで返信文を作成する：

   **修正した場合：**

   ```
   修正しました。{具体的な修正内容の要約}
   {該当コミットのSHA（短縮）}
   ```

   **対応しない場合：**

   ```
   ご指摘ありがとうございます。確認しましたが、以下の理由から現在の実装を維持します。
   {対応しない理由を具体的に説明}
   ```

4. **返信の投稿**
   各コメントに `gh api` で返信を投稿する。**レビューコメントへの返信**は「新規コメント作成」ではなく **replies エンドポイント**を使う：

   ```
   - `<comment_id>` はコメント一覧の各要素の `id`（数値。例: `2890156516`）。`in_reply_to` は使わない。
   - 改行を含む返信は `-f body="1行目\n2行目"` のように `\n` で渡す。
   gh api -X POST repos/<owner>/<repo>/pulls/<番号>/comments/<comment_id>/replies \
     -f body="返信内容"
   ```

5. **実行前の確認**
   全返信内容を一覧で表示し、投稿前にユーザーの承認を得る：

   | #   | コメントの要約 | 対応            | 返信内容 |
   | --- | -------------- | --------------- | -------- |
   | 1   | ...            | 修正 / 対応不要 | ...      |

   **ユーザーが承認するまで投稿しないこと。**
